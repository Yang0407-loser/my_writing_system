"这本小说的文字像钝刀割肉。多用短句，每段不超过三行，形容词能省则省。情感不要喊出来——压在冰面以下，让读者从动作和对话的缝隙里去感受。整体节奏前松后紧，到第三章开始加速。"

核心洞察：LLM 在自然语言上训练，不是参数表上训练。{"emotion_intensity": 75} 对生成质量几乎无影响，但"情感压在冰面以下"能精确塑造文风。二次 LLM 调用仅 800 tokens 成本，注入效果天壤之别。

关键设计：规则层处理 ~90% 明面矛盾（零 LLM 成本），LLM 层处理语义模糊。矛盾以 warning 注入而非阻断——宁可误报让 Writer 自主裁决，不可漏报导致正文矛盾。ENABLE_WORLD_STATE=false 可关闭整个模块。

为什么不在正文里用 [交接笔记] 标记？标记会残留在最终输出中，正则切分不可靠（嵌套、转义、格式变异）。独立 LLM 调用分离提取：正文干净无残留，提取 prompt 可独立调优，交接数据结构化存储可追溯。

设计 5 · Agent 角色哲学 — "笔记员"而非"裁判"
flowchart TD WR["Writer 生成正文"] --> CHK["一致性检查 · ContinuityEditor · Reviewer"] CHK -->|"发现问题"| NOTE["记录事实 · 标注提醒
生成 warning / 修正建议"] NOTE --> INJ["注入 Writer Prompt
'前文提到X, 请注意Y'"] INJ --> DECIDE["Writer 结合大纲+伏笔上下文
自主判断是否修正"] DECIDE -->|"确认矛盾"| FIX["执行修正"] DECIDE -->|"判断为伏笔/假动作"| SKIP["跳过 · 标注原因"] FIX --> USER["用户最终审批"] SKIP --> USER style NOTE fill:#7c8aff,color:#fff style DECIDE fill:#f59e0b,color:#000,stroke-width:2px style USER fill:#4ade80,color:#000

核心哲学：下游 Agent 只做"笔记员"——记录事实、标注提醒，不做强制裁决。矛盾检测以 warning 注入而非阻断；修正建议供参考而非强制执行。最终决策权始终保留给用户——Agent 是创作者的助理，不是替代判断的裁判。

设计 6 · 架构决策：串行 vs 并行 · 黑板 vs 消息传递
flowchart TD subgraph WHY_SERIAL["为什么串行?"] DEP["Agent 间强依赖
Phase 3 依赖前 5 项全部输出"] --> BOT["瓶颈在 Writer (80%+ 耗时)
并行其他 9 个不改善整体延迟"] BOT --> SAFE["串行 = 天然写屏障

串行不是偷懒——是架构分析后的选择。10 个 Agent 中 Writer 独占 80%+ 耗时，并行化其余 9 个对整体延迟改善微乎其微，但引入并发竞态的代价巨大（加锁、事务、一致性）。黑板不是简单的 KV 存储——是 Agent 通信协议的架构选择。点对点消息传递 = 每个 Agent 要知道所有上下游 = 强耦合；黑板 = Agent 只和黑板交互 = 低耦合、可独立替换。
